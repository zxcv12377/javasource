인터페이스
 - 추상 클래스(단 추상화 정도가 높다)
 - 일반 메서드, 멤버변수 가질 수 없음
 - 상수, 추상 메서드만 가짐 (static, default를 사용하면 가능)
 - 부모 역할로만 사용(자손 클래스에서 추상 메서드를 반드시 구현하도록 강요)
 - 인터페이스 상속 - 인터페이스로만 상속받을 수 있음 / 다중상속
 - 장점 : 표준화가 가능하다. / 서로 관계없는 클래스들에게 관계를 맺어줄 수 있다.

단일 상속
 - 클래스는 단일상속만 가능 
 - class A extends B{}의 형태

추상 클래스
 - abstract 키워드 사용
 - 추상메서드 사용
 - 인스턴스 생성 불가 (new 사용 불가)
 - 부모 역할로만 사용(자손 클래스에서 추상 메서드를 반드시 구현하도록 강요)
 - 추상 메서드의 Overriding이 강제됨
 - 추상화 : 클래스간의 공통점을 찾아내서 공통의 조상을 만드는 작업
 - 구체화 : 상속을 통해 클래스를 구현, 확장하는 작업

다형성(Polymorphism)
 - 상속과 관련이 있음
 - 여러가지 형태를 가질 수 있음
 - 한 타입의 참조변수로 여러 타입의 객체를 참조
 - 조상 타입의 참조 변수로 자손 클래스의 인스턴스를 참조
 - 참조 변수의 타입에 따라 사용할 수 있는 멤버의 갯수가 달라진다
   (부모에 선언된 멤버만 접근 가능 - 단, 오버라이딩 된 메소드가 있다면
   자손 클래스의 메소드가 실행됨)
