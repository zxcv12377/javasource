Thread
- 프로세스 : 현재 실행중인 프로그램
- 쓰레드 : 하나의 프로세스 안에 하나 이상의 쓰레드 사용
           실제 프로그램이 수행되는 최소 작업 단위 

 멀티 쓰레드 작성
 1) Thread 클래스 사용
 2) Runnable 인터페이스 사용

 실행 대기상태에 있는 쓰레드를 스케쥴링에 의해 번갈아 가면서 실행
 CPU 싱글코어 : 쓰레드 우선순위에 따라 우선순위가 높은 쓰레드가 
                실행할 기회를 더 많이 받음
     쿼드코어 : 한번에 4개의 쓰레드가 같이 실행


동기화 메소드와 동기화 블록
- 멀티쓰레드 프로그램에서는 쓰레드들이 객체를 공유해서 작업하는 경우가 있음
- 공유해서 사용하는 경우 한 쓰레드가 먼저 사용중이라면 다른 쓰레드의 접근을
  막아야 함 (임계 영역)
- 임계 영역 지정 => sychronized
